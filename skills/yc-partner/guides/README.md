# Guides

Maintained Markdown guidance built from the processed source files.

Update these files as resources change. Automated passes can refresh source-grounded sections, and optional human-curated notes can be added in the same guide files when they improve the guidance. Do not require a separate curated file for every generated update.

Guides may cite bundled source files they depend on and should keep official YC guidance separate from external examples and inference. Runtime answers should translate those internal citations into public URLs for founders.

- `application-review.md` - maintained review rubric and source-grounded review passes.
- `application-research.md` - public URL, founder profile, company profile, website/demo, and YC-directory research workflow for deeper application reviews.
- `application-hygiene.md` - concise required-field, typo, link, video, demo, and directness checklist for application reviews.
- `founder-advice-notes.md` - short source-grounded notes, tips, and application/interview advice selected per founder, stage, profile, question, or review issue.
- `office-hours.md` - source-grounded office-hours mode for founder obstacles, worries, doubts, and advice requests.
- `startup-operating-notes.md` - office-hours-friendly operating notes for customer learning, MVPs, launches, focus, burn, hiring, team building, and AI startup questions.
- `investor-meeting-prep.md` - source-grounded investor meeting, seed fundraising, runway, burn, and fundraising-story preparation.
- `question-bank.md` - reusable source-grounded questions for office hours, user discovery, YC interview prep, investor prep, hiring, and operating check-ins.
- `rejection-prep.md` - rejection, reapplication, acceptance-rate, myth-busting, and post-rejection review guidance.
- `interview-prep.md` - maintained YC interview-prep guidance.
- `fun-facts.md` - curated source-linked fun facts and YC insider/stat nuggets for one lightweight `Fun Fact` item in broad application reviews.
