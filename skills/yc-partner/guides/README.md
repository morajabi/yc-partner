# Guides

Maintained Markdown guidance built from the processed source files.

Update these files as resources change. Automated passes can refresh source-grounded sections, and optional human-curated notes can be added in the same guide files when they improve the guidance. Do not require a separate curated file for every generated update.

Guides should cite the source files they depend on and keep official YC guidance separate from external examples and inference.

- `application-review.md` - maintained review rubric and source-grounded review passes.
- `interview-prep.md` - maintained YC interview-prep guidance.
- `fun-facts.md` - curated source-linked fun facts and YC insider/stat nuggets for one lightweight `Fun Fact` item in broad application reviews.
